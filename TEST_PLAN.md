# 국립박물관 메인 페이지 웹 테스트 계획서 (TEST_PLAN.md)

본 문서는 국립박물관 프로젝트의 메인 페이지 기능 및 웹 접근성을 검증하기 위한 자동화 테스트 계획서입니다. 모든 테스트 시나리오는 **Given-When-Then** 구조를 엄격히 준수하여 정의되었습니다.

---

## 📋 1. 테스트 목적 및 범위
* **목적**: 사용자 경험(UX) 극대화, 다중 디바이스 대응(반응형), 그리고 스크린 리더 및 장애인 사용자를 위한 웹 접근성 표준 준수 검증.
* **범위**: 메인 페이지(`http://localhost:5173/museum/`) 내 주요 UI 요소, 내비게이션, 모바일 반응형 헤더, 그리고 종합 웹 접근성 진단.

---

## 🧪 2. 테스트 시나리오 (Given-When-Then)

### 시나리오 1: 메인 페이지 종합 웹 접근성 검증 (Axe-Core)
* **Given**: 로컬 개발 서버가 구동 중이고 사용자가 크롬(Chromium) 브라우저로 국립박물관 메인 페이지([index.html](file:///d:/SRC/VIBE/museum/index.html))에 접속했을 때
* **When**: Playwright 및 `@axe-core/playwright`를 통해 WCAG 2.0/2.1 A, AA 표준 및 Best Practice 규격에 따른 접근성 자동 스캔을 실행하면
* **Then**: 페이지 내에 심각한 접근성 위반 사항(예: 색상 대비 오류, Heading 계층 오류 등)이 없어야 하며 위반 항목 수가 `0`개여야 한다.

### 시나리오 2: 글로벌 내비게이션 바(GNB) 메뉴 노출 및 링크 유효성 검증
* **Given**: 사용자가 데스크톱 해상도(가로 1280px 이상)로 메인 페이지에 진입했을 때
* **When**: 상단 헤더 영역의 메뉴(예: 전시, 예약, 박물관 소개 등)를 확인하고 각 메뉴의 링크 속성을 스캔하면
* **Then**: 모든 주요 메뉴가 화면에 깨짐 없이 올바르게 렌더링되어야 하며, 각 메뉴 항목은 유효한 타겟 링크(`href`)를 포함하고 있어야 한다.

### 시나리오 3: 모바일 반응형 헤더 및 토글 메뉴 동작 검증
* **Given**: 사용자가 모바일 뷰포트(가로 375px) 크기로 메인 페이지에 진입하여 상단 헤더에 모바일 햄버거 메뉴 버튼이 노출되었을 때
* **When**: 사용자가 햄버거 메뉴 버튼을 탭(클릭)하면
* **Then**: 모바일 네비게이션 드로어(오버레이 메뉴)가 부드럽게 화면에 나타나며, 다시 닫기 버튼을 누르면 메뉴가 정상적으로 사라져야 한다.

### 시나리오 4: 메인 배너 슬라이더(온라인 전시) 인터랙션 및 인디케이터 동기화 검증
* **Given**: 사용자가 온라인 전시 배너 내비게이션 영역의 이전/다음 제어 버튼 및 페이지 인디케이터가 렌더링된 메인 페이지를 보고 있을 때
* **When**: 사용자가 '다음 전시'(`>`) 버튼을 클릭하면
* **Then**: 배너 슬라이드가 다음 화면으로 전환되어야 하며, 현재 페이지 인디케이터 번호가 `1`에서 `2`로 즉각 업데이트되어 동기화되어야 한다.

## 🛠️ 3. 테스트 수행 환경 및 실행 도구
* **E2E/자동화 프레임워크**: [playwright.config.js](file:///d:/SRC/VIBE/museum/playwright.config.js) 설정 기반의 Playwright Test
* **웹 접근성 엔진**: `@axe-core/playwright`
* **대상 브라우저**: Chromium (Desktop Chrome), WebKit (Desktop Safari), Firefox
* **테스트 구동 명령어**:
  ```bash
  npm run test:accessibility
  ```
